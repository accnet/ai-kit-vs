# Artifact Contracts

The control plane is the only writer of workflow lifecycle state. Role plugins exchange versioned JSON artifacts under `.ai-work/workflows/<workflow-id>/artifacts/`.

- `assignment`: generated by the runner with the role, actor, task, attempt, and scoped context.
- `plan`: generated by a planner with a goal and atomic task definitions. The State Manager validates it before adding tasks.
- `result`: generated by an executor with task, attempt, status, summary, commands, and changed paths.
- `qa`: generated by an independent QA plugin with status, summary, and verification commands.
- `review`: generated by an independent reviewer with `approve` or `changes-requested` and notes.

Every artifact has `version: 1`, `kind`, `workflow_id`, and `actor`. Output must be written atomically to the supplied path. The runner rejects invalid JSON, mismatched workflow/task/actor identities, output outside the workflow workspace, and attempts that are no longer claimed.

For `changes-requested`, the State Manager records the review, replaces the reviewed task, creates a remediation task, and rewires its downstream dependencies. No plugin mutates the queue directly.
